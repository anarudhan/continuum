# Security Policy

## Reporting Security Issues

Security issues should be reported to pinches-tapioca0v@icloud.com.

Do NOT open public issues for security vulnerabilities.

We will respond within 48 hours and work with you to verify and fix the issue.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅ Yes    |
| Older   | ❌ No     |

## Security Design Principles

1. **Secure by default** — Works safely out of the box
2. **Zero friction** — Security never blocks legitimate agent workflows
3. **Transparent** — Users can see protection, don't need to manage it
4. **Fail open** — If security system fails, agents keep working
5. **Minimal config** — One env var (API key), everything else automatic
